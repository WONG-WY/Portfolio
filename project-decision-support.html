Project

Full-Stack School Decision-Support Tool

Vue.js Node.js Express.js MongoDB Auth0 Chart.js JWT

A full-stack web platform that transforms scattered government school data into an interactive, comparable, and visualized decision-making tool for Hong Kong families.

Motivation

Choosing a secondary school in Hong Kong means digging through government websites, PDF tables, and disconnected data sources — then manually comparing schools on paper. No existing platform lets families filter, rank, visualize, and compare schools in one place with a clean interface. This project was built to change that: pulling real government data into a single platform where the decision becomes clearer.

What I Built

  • api
    Real-time school data

    Integrated Hong Kong Government public APIs to fetch live school information and coordinates, keeping data accurate without manual updates.

  • bar_chart
    Interactive dashboard

    Built a Chart.js visualization layer showing school distributions across type, gender, district, and language — turning raw data into readable charts.

  • tune
    Smart filtering and ranking

    Users can filter schools by multiple criteria simultaneously, and rank results by DSE performance or calculated distance from any given address using the Haversine formula.

  • compare
    Side-by-side comparison

    Two schools can be placed directly side by side, with advantages automatically highlighted — eliminating the need to switch between tabs and take notes manually.

  • lock
    Secure authentication with custom UI

    Implemented Auth0 ROPC flow to protect APIs with JWTs while keeping a fully custom-designed interface, avoiding Auth0's default login page.

Technical Highlight — Dual-Write Architecture

Auth0's free tier does not allow direct connections to custom databases. This created a problem: users registered in MongoDB would not exist in Auth0, breaking JWT-based API protection.

  • Solution:

    A dual-write mechanism built into the Express.js registration endpoint — every new user is written to MongoDB first, then immediately created in Auth0 via the Management API.

  • Result:

    MongoDB remains the single source of truth for all user data, while Auth0 handles token issuance exclusively. No third-party login page. No compromise on security.
