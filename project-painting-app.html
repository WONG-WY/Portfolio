Project

Real-Time Synchronized Painting App

Java Sockets Multithreading Synchronization TCP/IP Network Programming

A single-player drawing board transformed into a live multiplayer canvas, built from the ground up with a custom Java server handling concurrent connections, thread safety, and real-time state synchronization.

Motivation

The starting point was a single-player drawing application provided as a base. The challenge: turn it into a real-time multiplayer experience where every stroke appears instantly on all connected clients — without conflicts, without lag, without data loss.

What I Built

  • dns
    Custom Java socket server

    Built from scratch to accept and manage multiple client connections simultaneously over TCP.

  • account_tree
    Multithreaded architecture

    Dedicated thread per client isolates connections, preventing one client's actions from blocking others.

  • broadcast_on_home
    Real-time canvas broadcast

    Every drawing action is serialized and pushed to all connected clients instantly, keeping all canvases in sync.

  • router
    Network configuration

    Resolved firewall and port configuration issues to enable stable external connections across different network environments.

  • sync
    Concurrent synchronization

    Synchronization logic ensures draw events from multiple simultaneous clients are processed in correct order without race conditions.

Technical Highlight — Concurrent Synchronization

The hardest problem was not connecting multiple clients — it was keeping them in sync when several draw simultaneously. Without proper synchronization, two clients drawing at the same moment would produce conflicting canvas states, with one client's strokes overwriting another's or arriving out of order.

  • Solution:

    A thread-safe message queue on the server: all incoming draw events are serialized into a single ordered stream before broadcasting to all clients.

  • Result:

    Combined with Java's synchronized blocks to protect shared canvas state, every client sees an identical, consistent canvas regardless of network timing or the number of concurrent users.
