Project

Real-Time Synchronized Painting App

Multiplayer collaborative canvas with live synchronization

Java WebSockets Multithreading

Overview

Transformed an existing single-player painting application into a real-time multiplayer experience. The backend server manages concurrent connections and synchronises canvas state across all connected clients instantly using Java Sockets.

What Was Built

  • Engineered a backend server to convert a single-player app into a real-time multiplayer canvas
  • Utilised Java Sockets to efficiently handle concurrent client connections and data synchronisation
  • Led server-side development and system integration to ensure timely project completion

Key Outcomes

  • Low-latency canvas sync across multiple simultaneous users
  • Robust concurrent connection handling via multithreaded socket management
  • Successful delivery of multiplayer feature within project timeline
